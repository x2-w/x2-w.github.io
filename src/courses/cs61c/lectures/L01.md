## 计算机系统结构角度思考

> [!IMPORTANT]
> 本课程不是关于C语言编程的


这是一门 **硬件与软件的接口（Hardware-Software Interface）** 的课程

软件开发人员需要理解底层硬件，才能实现系统更高性能。

而 C 语言比 Python/Java 等高级编程语言更接近硬件，C 允许程序员显式利用硬件并行性，实现高性能程序。

因此学习计算机系统架构的底层细节非常重要

> [!NOTE]
> 【Q】为什么要通过 `CS61C` 课程学习计算机底层架构原理？
>
> 【A】高级语言隐藏了硬件细节, 但性能优化需要理解硬件, C语言更接近机器， 因此能更好地控制性能


## 计算机体系结构中伟大思想

计算机体系结构中6个伟大思想

1. Abstraction => 抽象（表示层 / 解释层）
2. Moore’s Law => 摩尔定律
3. Principle of Locality / Memory Hierarchy => 局部性原理 / 存储层次结构
4. Parallelism => 并行性
5. Performance Measurement & Improvement => 性能评估与提升
6. Dependability via Redundancy => 通过冗余提高可靠性

### 伟大思想 #4：并行性

### 伟大思想 #5：性能评估与提升

核心思想: 计算机性能必须 **可量化**, 而不是凭感觉优化

### 伟大思想 #6：通过冗余提高可靠性

核心思想：计算机系统中硬件一定会出错， 因此需要通过冗余提高系统可靠性

## 关于这么课程，需要了解什么？

学习计算机体系结构中的六大伟大思想，通过并行性实现高性能编程，而不仅仅是学习 C 语言。
